## 今回の目的

確認用 demo ページで、選択中のメモグリッドに対して共有相手を管理できるようにします。

具体的には、demo ページに次の操作を追加してください。

- 選択中のメモグリッドの共有相手一覧を取得する
- `username` を入力して共有相手を追加する
- 共有相手一覧から共有を解除する

共有相手管理 API は作成者だけが利用できます。
そのため、共有されたユーザーで demo ページを操作している場合、共有相手の一覧取得・追加・削除は `404 Not Found` になる想定です。

## 今回やること

- demo ページに共有相手管理用の UI を追加する
- 選択中のメモグリッド ID を使って共有相手一覧 API を呼ぶ
- `username` 入力欄から共有相手追加 API を呼ぶ
- 共有相手一覧に削除ボタンを表示し、共有相手削除 API を呼ぶ
- API 成功後に共有相手一覧を再取得する
- API エラー時に、既存 demo ページの方針に合わせてエラー内容を表示する
- 必要に応じて `MapDemoViewTests` に表示確認を追加する
- 必要に応じて `README.md` の demo ページ確認手順を更新する

## 対象 API

### 1. 共有相手一覧 API

```text
GET /api/maps/areas/{area_id}/shares/
```

選択中のメモグリッドの共有相手一覧を取得します。

成功時のレスポンス例:

```json
{
  "area": {
    "id": 1,
    "name": "東京駅周辺"
  },
  "shares": [
    {
      "id": 1,
      "area": 1,
      "user": {
        "id": 4,
        "username": "otheruser"
      },
      "created_at": "2026-05-22T10:00:00+09:00"
    }
  ]
}
```

共有相手がいない場合:

```json
{
  "area": {
    "id": 1,
    "name": "東京駅周辺"
  },
  "shares": []
}
```

### 2. 共有相手追加 API

```text
POST /api/maps/areas/{area_id}/shares/
```

選択中のメモグリッドに共有相手を追加します。

リクエスト:

```json
{
  "username": "otheruser"
}
```

成功時のレスポンス例:

```json
{
  "share": {
    "id": 1,
    "area": 1,
    "user": {
      "id": 4,
      "username": "otheruser"
    },
    "created_at": "2026-05-22T10:00:00+09:00"
  }
}
```

### 3. 共有相手削除 API

```text
DELETE /api/maps/areas/{area_id}/shares/{share_id}/
```

指定した共有関係を削除します。

成功時は `204 No Content` です。

## UI 方針

demo ページ内に、共有相手管理用のセクションを追加してください。

表示名は、ユーザー向けに分かりやすくするため、次のような文言を使ってください。

```text
共有相手管理
共有相手一覧を取得
共有相手を追加
共有を解除
共有相手 username
```

内部名として `MapArea` や `MapAreaShare` を使うのは構いませんが、画面表示ではできるだけ次の呼称を使ってください。

- 個人用の `MapArea`: `メモグリッド`
- 共有された `MapArea`: `共有メモグリッド`
- 共有先ユーザー: `共有相手`

## UI の具体要件

### 共有相手管理セクション

既存のメモグリッド一覧・GridCell 表示の流れを壊さない位置に、共有相手管理セクションを追加してください。

最低限、次を置いてください。

- 選択中のメモグリッド名または ID の表示
- `共有相手一覧を取得` ボタン
- `共有相手 username` 入力欄
- `共有相手を追加` ボタン
- 共有相手一覧表示エリア
- 各共有相手の `共有を解除` ボタン
- 成功・エラーメッセージ表示エリア

### メモグリッド未選択時

メモグリッドが選択されていない状態で共有相手一覧取得・追加・削除をしようとした場合は、API を呼ばず、画面上に分かりやすいメッセージを出してください。

例:

```text
先にメモグリッドを選択してください。
```

### 共有相手一覧表示

共有相手一覧では、最低限次を表示してください。

- 共有相手の `username`
- 共有関係の `id`
- `共有を解除` ボタン

`email` は表示しないでください。

共有相手が 0 件の場合は、空のままにせず、次のようなメッセージを表示してください。

```text
共有相手はまだ登録されていません。
```

### 共有相手追加

`username` 入力欄が空の場合は、API を呼ばず、画面上にメッセージを表示してください。

例:

```text
共有相手 username を入力してください。
```

追加成功後は、入力欄を空にし、共有相手一覧を再取得してください。

### 共有相手削除

削除ボタンを押したら、対象の `share_id` を使って次を呼び出してください。

```text
DELETE /api/maps/areas/{area_id}/shares/{share_id}/
```

削除成功後は、共有相手一覧を再取得してください。

確認用 demo ページなので、削除前の確認ダイアログは必須ではありません。
ただし、実装が簡単で既存 UI の邪魔にならない場合は、`confirm()` を使っても構いません。
