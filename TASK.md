# TASK: 地図プレビューに色の濃さスライダーを追加する

## 目的

`/maps/<area_id>/` の地図プレビューに、グリッド面の半透明色を調整できる「色の濃さ」スライダーを追加する。

透明度を直接指定するのではなく、現在の基本透明度に倍率をかける方式にする。

## 対象ファイル

- `maps/templates/maps/grid_detail.html`
- `maps/static/maps/js/grid-detail.js`
- `maps/static/maps/css/site.css`

## やること

- 地図プレビュー付近に「色の濃さ」スライダーを追加する
- スライダー範囲は `0〜100`、初期値は `50` にする
- `50` のとき、現在のグリッド透明度をそのまま使う
- `0` に近いほど薄く、`100` に近いほど濃くする
- 通常セル、未分類セル、選択中セルの透明度差は維持する
- 実際の透明度は「基本透明度 × 倍率」で計算する
- 濃くなりすぎないよう、透明度には上限を設ける
- スライダー変更時に地図上のグリッド表示を更新する
- 採点・選択・再取得ロジックは変更しない

## やらないこと

- 透明度を固定値で直接指定する方式
- localStorage などへの設定保存
- スコア色分けルールの変更
- スコアラベル表示の変更
- 採点ロジック変更
- 選択仕様変更
- API / model / migration 変更
- demo.js / demo.css の変更
- `memo.md` 更新

## 注意

- 初期表示が現在の見た目と同じになること
- 色を薄くしても選択中マスが分かること
- 色を濃くしても地図タイルが完全に見えなくならないこと
- 採点後・グリッド再取得後もスライダー値が反映されること
- 確認コマンドやテストコマンドは実行しないこと

- 変更したファイル
- スライダーの配置場所
- 倍率計算の方法
- 透明度の上限
- 採点後・再取得後も反映される仕組み
- 既存の選択・採点・再取得処理を変更していないこと
- 確認コマンドやテストコマンドを実行していないこと
